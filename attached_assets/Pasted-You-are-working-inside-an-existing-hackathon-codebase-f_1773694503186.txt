==================================================
STAGE-BY-STAGE LOGIC
==================================================

STAGE A: PROPOSALS
- Each founder creates exactly one startup proposal
- One proposal per founder only
- Proposal must include all structured fields
- Append transcript entry after each proposal
- Save founder_proposals artifact

STAGE B: CRITIQUES
- Each founder critiques the other 3 proposals
- That means 12 critiques total
- Each critique must include:
  - strengths
  - weaknesses
  - verdict
  - score
- Append transcript entries as critiques occur
- Save founder_critiques artifact

STAGE C: REVISIONS
- Each founder receives all critiques targeting THEIR proposal
- They revise their proposal exactly once
- Revised proposal must preserve the same core idea
- Revised proposal must include:
  - revisionSummary
  - changesMade
- Append transcript entries after each revision
- Save founder_revised_proposals artifact

STAGE D: VOTES
- Each founder votes on one revised proposal that is NOT their own
- Each vote must include rationale
- Append transcript entry
- Save founder_votes artifact

STAGE E: WINNER SELECTION
Use deterministic backend logic.

Selection logic:
1. Count votes for each revised proposal
2. Highest vote count wins
3. If tie, use aggregate critique score from critiques targeting that proposal’s originalProposalId
4. If still tied, use stable deterministic fallback:
   - market preference vote
   - then finance preference vote
   - then lexical sort by proposal id as a final deterministic tie-breaker

Do NOT use a final "AI picks winner" pass.

STAGE F: CANONICAL BRIEF SYNTHESIS
- Use the selected revised proposal
- Also pass in the critiques that originally targeted that proposal lineage
- Generate a worker-ready canonical brief
- Save founder_result artifact with everything
- Update run/company summary state so the frontend shows:
  - company name
  - tagline
  - summary / short description
- Mark founder phase complete
